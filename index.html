What is CATCH?
CATCH validates CMS publisher payloads against the TX OCA and IL AOIC community pipeline schemas. Paste a failed ACB payload into the Validate tab and run — each entity returns a pass/fail with plain-English translations, fix ownership, and recommended next steps. All runs are saved automatically to History / Export, and deduplicated across sessions in the Error Catalog tab.

Schema rules, valid entity types, and allowed values live in Schema Reference. The Error Library (⚙ Settings) lets the BIS team codify recurring error patterns with custom translations that appear inline in validation results.
What CATCH catches — TX OCA
Invalid entity type namese.g. di-texas-oca-court-charge (singular) — does not exist in the contract
County registration gapsOdyssey Source ID is registered for ~105 TX counties — real counties outside that set are flagged with a D&I ticket escalation note
Wrong field namese.g. filing_statute_citation → statute_citation (OCA-010)
Type violationsFields that must be number|null submitted as strings
Additional propertiesFields not in the contract schema
Enum violationse.g. plea_type "Guilty" must be "G - Guilty"
Missing required fieldscounty, publisher, and recordid required on all entity types
What CATCH catches — IL AOIC
Invalid entity type namesMust match one of the 37 recognized AOIC types exactly (case-sensitive)
Missing instanceid / localidBoth required on all AOIC entity types
Unregistered CMS vendorname field must match an AOIC-registered CMS vendor name exactly
Type violationsNumber fields must not be quoted strings
Enum violationsList-type fields must match allowed values exactly
Missing required fieldsTrial Court and Reviewing Court entities have schema-required fields
Known Limitations
AEP branch validation (TX)AEP evaluates Odyssey payloads against the wrong oneOf branch. CATCH resolves Source IDs correctly and does not replicate these false positives.
Odyssey county listThe ~105-county restriction was extracted from real ACB error responses — confirm with OCA if unexpected county failures appear.
Schema versionsTX OCA v3.0.0 · IL AOIC v3.1.0 — both embedded March 2026. Contact BIS if updated schemas are needed.
IL Pretrial / Probation required fieldsThose dictionaries don't explicitly mark fields as required — validation focuses on enum and type checks for those domains.
Error Library matchingLibrary entries match on field + value or field name. Complex multi-field patterns require multiple entries or static rule additions.

v2.5 Latest March 2026 · BIS Engineering
⬆ New — Timestamp Tracking
Envelope submission time The OriginalTimestamp Unix millisecond value is now converted to a human-readable local time and displayed in the summary bar as ⬆ submitted, so you can immediately see when the AEP received the envelope.
CATCH analysed-at time Each validation run now records the exact time CATCH processed the envelope, shown as ⚡ analysed in the summary bar and History tab — distinct from the submission timestamp so lag is always visible.
Error Log timestamps Auto-logged Error Log entries now include the envelope's ⬆ submitted time alongside CATCH's analysis date.
CSV / JSON exports History exports now include separate columns: Analysed At (CATCH), Envelope Submitted At (AEP), and OriginalTimestamp (Unix ms).
◈ Improved — Error Catalog
Envelope log column The Error Catalog table now includes an Envelopes column — every envelope ID that contributed to each error pattern is tracked and displayed (up to 3 visible, full list on hover).
✦ New — UX Improvements
EnvelopeId override Moved from the History tab to the Validate pane footer — now visible and accessible at the point of use, before running.
Entity type in results Result card headers now show the entity type inline (e.g. di-texas-oca-court-case-status) so you don't have to expand the metadata to identify which entity failed.
Drag-and-drop input Drop a .json file directly onto the input pane to load it — no copy-paste required.
Entity count preview The input pane header shows a live entity count as you type or paste — with a red invalid JSON indicator if the payload is malformed before you even run.
Tab badges The History and Error Catalog tabs now display live numeric badges — History shows total run count, Error Catalog shows total logged error count.
History filter clear A ✕ button appears next to the history filter input when text is entered, clearing it in one click.
Run button state The Run Validation button now shows ⏳ Running… and disables during processing — prevents double-clicks on large payloads and provides clear feedback.
Error Log search Free-text search added to the Error Log toolbar — search across envelope IDs, cause text, field names, counties, and BIS notes simultaneously.
Colour legend A ? legend button on the summary bar opens a popover explaining every colour used across the tool — timestamp colours, validation result colours, Error Catalog category badges, and History envelope badges.
