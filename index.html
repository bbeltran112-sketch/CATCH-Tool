User Guide
Review payload quality, understand errors, and confirm the contract in one place.
CATCH is designed to help teams validate a payload, translate the validator output into readable language, confirm the active schema, and track recurring issues over time. It supports both day-to-day triage and leadership reporting without changing the underlying validation engine.
Best For
Payload review
Validate one submission, inspect each failing entity, and understand what the validator is saying.
Schema Source
Workbook + Contract
Compare the approved workbook against the runtime contract using the schema toggle in the Event Type view.
Reporting
History + Trends
Review saved runs, export results, and track recurring patterns across fields, publishers, and entity types.
Refinement
Library-driven
Improve recurring plain-English translations over time by refining errors directly from findings and trend details.
Single for one payload History for saved envelopes Trends for recurring patterns Event Type for schema confirmation

CATCH validates CMS publisher payloads against active TX OCA and IL AOIC contract rules, then translates the validation results into clearer business language. It is designed to help project teams review a single payload, save those runs over time, and monitor recurring issues across publishers, fields, and entity types.
Quick Start
1. Validate a payloadOpen Single, paste a payload, and review the technical errors and plain-English translations returned for each entity.
2. Confirm the schemaClick the Event Type card to open schema review. Use the Merged, Workbook, or Contract toggle to compare views.
3. Review saved runsOpen History to inspect prior envelopes, export results, or restore browser-backed history.
4. Watch recurring issuesOpen Trends to identify repeated validation patterns, publisher concentration, movement, and aging.
5. Refine translationsUse Refine this error to improve plain-English messages and save them into the library for future runs.
Main Workspaces
SingleBest for one payload at a time. This is where users validate, review findings, inspect event type context, and refine translations.
BatchBest for many payloads at once. Use this workspace to process a queue, review high-level results, and reopen any one payload back into Single for deeper inspection.
HistoryBest for reviewing past runs. Use it to inspect saved envelopes, export results, and manage local browser backup and restore.
TrendsBest for recurring-pattern analysis. Use it to track repeat issues, publisher concentration, issue mix, health signals, and row-level drill-in.
Schema Review
MergedBest general-use view. Combines the approved workbook metadata with active contract rules.
WorkbookBest for field definitions, required-field intent, workbook priority, and a spreadsheet-style schema reference.
ContractBest for runtime structure and raw contract logic. Use this when you need the machine-facing view of the schema.
Full schemaExpand Full schema inside an entity card to inspect the raw JSON contract.
What CATCH Flags
Missing fieldsRequired fields that were not included in the payload.
Type mismatchesValues sent in the wrong format, such as text in quotation marks where a number is expected.
Invalid valuesValues that do not match the approved list for a contract field.
Extra fieldsFields that are not part of the contract for the entity type.
Wrong field namesFields that appear to be close to a contract field but do not match the approved name.
Entity and publisher issuesInvalid entity types, publisher mismatches, county enum issues, and other market-level validation problems.
Data Storage and Export
Browser storageHistory and refinement data are stored locally in the browser unless exported or cleared.
ExportsUse Workbook, JSON, and Health Report exports from History and Trends to share results outside the browser.
Backup and restoreUse Backup and Restore to preserve browser-stored history across machines or sessions.
Who Should Use This View
Project teams should use CATCH to validate a single payload, confirm the schema, and understand what failed.
Senior leadership should use Trends and Health reporting to monitor recurring issue concentration, movement, and publisher pressure.
Support and TPM teams should use refinement and schema review to improve translation quality and maintain reusable guidance over time.
